# Role Requirements: ML Platform Engineer

**Level:** 2.5
**Experience Level:** 3-6 years in software engineering, DevOps, or ML infrastructure, with at least 1-2 years in platform or tooling development
**Market Demand:** High - Growing demand as companies build internal ML platforms. Tech companies, financial services, and large enterprises actively hiring. Smaller pool of candidates with both strong software engineering and ML platform expertise. Critical role for organizations scaling ML capabilities.
**Salary Range (US, 2025):** $145,000 - $296,000 (median $180,000)

## Core Responsibilities
- Design and build internal ML platforms for data science and ML engineering teams
- Develop self-service tools and APIs for model training, deployment, and monitoring
- Create and maintain SDKs and client libraries for ML infrastructure
- Build feature stores and data versioning systems
- Implement workflow orchestration systems for ML pipelines
- Design and maintain experiment tracking and model registry systems
- Develop internal developer portals and documentation for ML platform
- Optimize platform performance and reliability for diverse ML workloads
- Create automation tools to reduce friction in ML development lifecycle
- Gather requirements from ML practitioners and translate to platform features
- Maintain backward compatibility and versioning for platform APIs
- Build platform observability and usage analytics
- Support users of ML platform and address feature requests
- Integrate third-party ML tools into unified platform experience

## Technical Skills
### Required
- Strong software engineering skills in Python and one other language (Go, Java, Scala)
- API design and development expertise (REST, gRPC)
- Kubernetes and cloud-native application development
- Experience with ML frameworks and their integration
- SDK and library development experience
- Database design and data modeling
- Workflow orchestration tools (Airflow, Prefect, Kubeflow)
- Understanding of ML lifecycle and MLOps practices
- Platform engineering principles and patterns
- CI/CD for platform components

### Preferred
- Experience with feature stores (Feast, Tecton, SageMaker Feature Store)
- Knowledge of internal developer platforms (Backstage, Humanitec)
- Model serving frameworks (Seldon, KServe, BentoML)
- Stream processing (Kafka, Flink)
- Metadata management and data catalogs
- Experience with Apple ML infrastructure or similar large-scale platforms
- GraphQL API development
- Platform authentication and authorization (OAuth, RBAC)
- Multi-tenancy architecture design
- Developer experience (DX) optimization

## Soft Skills
- Strong user empathy and customer-focus
- Excellent communication for gathering requirements
- Collaboration with diverse technical stakeholders
- Product thinking for platform features
- Technical writing and documentation
- Teaching and enablement
- Balancing platform flexibility with usability
- Stakeholder management
- Iterative development mindset
- Listening and incorporating feedback

## Prerequisites
- Bachelor's degree in Computer Science or related field; Master's beneficial
- Strong software engineering background with production systems experience
- Understanding of ML development workflows and pain points
- Experience building developer-facing tools or platforms
- Knowledge of cloud platforms and distributed systems
- Experience with API design and SDK development

## Day-to-Day Activities
- Developing new features for ML platform based on user feedback
- Meeting with ML teams to understand requirements and pain points
- Writing and maintaining SDKs and client libraries
- Building and improving platform APIs and services
- Creating documentation and tutorials for platform users
- Reviewing pull requests and conducting code reviews
- Debugging platform issues and providing user support
- Monitoring platform usage metrics and performance
- Designing and implementing platform architecture improvements
- Conducting platform demos and training sessions
- Evaluating third-party tools for platform integration

## Success Metrics
- Platform adoption rate across ML teams
- User satisfaction and Net Promoter Score (NPS)
- Reduction in time-to-deployment for ML models
- Platform uptime and reliability (99.9%+ SLA)
- API usage growth and engagement metrics
- Developer productivity improvements (measured via surveys)
- Number of models deployed through platform
- Platform documentation quality and completeness
- Reduction in support tickets through self-service features

## Common Challenges
- Balancing diverse requirements from different ML teams
- Maintaining platform stability while innovating rapidly
- Supporting legacy workflows while promoting best practices
- Managing platform complexity and cognitive load for users
- Keeping up with rapidly evolving ML tools and frameworks
- Building for scale while serving current needs
- Gathering and prioritizing feature requests effectively
- Ensuring platform security without hampering usability

## Key Technologies
- **Programming Languages:** Python (primary for ML ecosystem), Go (for platform services and tooling), Java or Scala (for data processing components), TypeScript (for web UI components), SQL (for data access)
- **ML Frameworks:** PyTorch, TensorFlow (integration knowledge), Scikit-learn, XGBoost, Hugging Face ecosystem, ONNX for model interchange, ML framework-agnostic design
- **Infrastructure:** Kubernetes (platform deployment), Cloud platforms (AWS, GCP, Azure), Container registries and artifact storage, Service mesh for inter-service communication, API gateways
- **DevOps Tools:** CI/CD platforms (GitHub Actions, GitLab CI, Jenkins), Infrastructure as Code (Terraform, Helm), GitOps (ArgoCD, Flux), Configuration management
- **MLOps Tools:** Kubeflow (Pipelines, Notebooks, KServe), MLflow (experiment tracking, model registry), Airflow or Prefect (workflow orchestration), DVC or Pachyderm (data versioning), Feature stores (Feast, Tecton), BentoML or Seldon Core (model serving), Weights & Biases, Neptune.ai (experiment tracking)
- **Monitoring & Observability:** Prometheus and Grafana (platform metrics), ELK or Loki (log aggregation), Distributed tracing (Jaeger, Zipkin), Usage analytics platforms, Custom metrics for platform adoption
- **Data Processing:** Apache Spark (for data pipelines), Kafka (for streaming), SQL databases (PostgreSQL, MySQL), NoSQL databases (MongoDB, Cassandra), Data catalogs (DataHub, Amundsen)
- **GPU Tools:** GPU resource management (for platform users), GPU scheduling and sharing, Awareness of GPU optimization for platform efficiency
- **Other Tools:** API development frameworks (FastAPI, Flask, gRPC), Authentication/Authorization (OAuth2, OIDC, RBAC), Developer portals (Backstage, Swagger/OpenAPI), Message queues (RabbitMQ, Redis), Object storage (S3, MinIO, GCS), Version control and branching strategies
